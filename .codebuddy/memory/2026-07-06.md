# 2026-07-06

## docker-stacks 项目 bug 修复

### 修复的 bug
1. **backup.sh — compose.yml 备注提取正则贪婪匹配**：`get_description()` 的 sed 捕获组 `(.+)` 贪婪匹配导致尾部 `====` 残留。改为两步剥离（先裁前面再裁后面）。
2. **backup.sh — printf ANSI 颜色码乱码**：`read -r -p` 和 `printf %s` 不解析 `\033` 转义码。`confirm()`、`press_enter()`、目标目录提示改用 `echo -en` + `read -r`；4 处 `printf %b` 修复了颜色变量输出。
3. **backup.sh — 备份路径解析缺失 app 前缀**：`$drel` 只有 `data/config`，确认和备份时漏了 `stacks/<name>/` 前缀，导致目录全部判定为"不存在"。
4. **backup.sh — init_state() 默认全选所有推荐目录**：用户期望按数字 = 添加选择，而非取消。改为默认不选任何目录，新增 `select_all_recommended()` 供 `[a] 全选推荐` 使用。
5. **backup.sh — 打包进度缺 app 名**：`正在打包 data/config` 改为 `正在打包 jellyfin/data/config`。
6. **backup.sh — set -e 导致输入数字后退出**：`toggle_app()` 中 `((rec_count++))` 在值为 0 时返回退出码 1，触发 `set -e` 退出脚本。简化切换逻辑：直接判断是否有任何选中项，避免算术递增。
7. **backup.sh — 交互 TUI 改为复选框风格**：用户反馈数字切换不直观。重写为光标导航 TUI：`↑↓/jk` 移动光标，`空格` 切换复选框，`[✓]` / `[ ]` 直观显示状态。
8. **backup.sh — "确认开始备份" 默认选项改为 Y**。
9. **backup.sh — 按空格切换第二个 app 时退出**：`toggle_app()` 的 else 分支 while 循环在最后一项为缓存目录时，`[[ "$is_cache" == "0" ]] && echo ...` 短路返回 1，退出码通过 while → 函数 → case 传播，在 `set -e` 下脚本直接退出。给 while 循环末尾加 `|| true` 兜底。
10. **backup.sh — TUI 增量更新消除闪烁**：改为首次全量绘制后，后续只重绘变化的行（`\033[<N>;0H\033[K` 跳行+擦除+重绘），不再全屏清屏。按箭头键只重绘两行，按空格只重绘当前行+摘要。
11. **backup.sh — 隐藏终端真实光标**：TUI 期间发送 `\033[?25l` 隐藏光标，退出/进入子菜单时发送 `\033[?25h` 恢复，避免屏幕上出现橘黄色 caret 方块。
12. **backup.sh — `[a]` 键改为全选/取消全选切换**：`has_any_selected` 判断，有则全不选，无则全选推荐。底部提示同步更新。

### 为 compose.yml 添加的备注标题
所有 9 个 compose.yml 添加了 `# =========== 中文名称 ===========  ` 格式的标题注释，以便 backup.sh 的 `get_description()` 提取显示。
