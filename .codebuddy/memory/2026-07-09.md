## HomeAssistant 重启死循环 + HASS.Agent 认证修复

### HA 权限问题
- 之前 `user: 1000:1000` 不适用于 HA 容器——s6 init 阶段以 root 创建文件，主进程降权后无法访问，导致 PermissionError 死循环重启
- 修复: 去掉 `user:` 指令，HA 内部自行管理权限
- 期间临时以 root 启动容器，`docker exec` chown 修复了大量 root 属主文件

### HASS.Agent 认证失败
- `192.168.1.160` 上 HASS.Agent 用旧 HA 实例的凭据连接全新 HA，每 5 秒报 Login attempt failed
- 解决办法:
  1. HA 侧: 用户头像 → 安全 → 长期访问令牌 → 创建 `HASS-Agent-Huang`
  2. HASS.Agent 侧: Server URI 改为 `http://192.168.1.146:8123`（不用 homeassistant.local）
  3. 重要: 先点保存再点测试，否则配置不持久化

### 还原功能增强
- 还原 TUI 交互升级: 方向键/jk 移动光标，空格选中/取消，a 全选/取消全选（统一切换）
- 还原前自动停止目标容器 → 解压 → 还原后自动启动
- 废弃并移除 `ds-restore`(restore.sh)，统一入口 `ds-backup`

## HA 集成安装 (米家 + 美的)

### HACS 安装
- 通过代理下载 HACS v2.0.5 zip，解压到 custom_components/hacs
- HA compose.yml 注入代理环境变量: HTTP_PROXY/HTTPS_PROXY/NO_PROXY (均引用 global.env)
- global.env NO_PROXY 补充 `.local`（mDNS 域名）
- Clash Party 不支持 `.local` 通配，需单独加具体域名如 `homeassistant.local`

### 小米 Xiaomi Home (官方)

- 解决方案: 关闭"自动"→ 手动设为 `http://192.168.1.146:8123`
