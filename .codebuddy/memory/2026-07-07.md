## backup.sh 修改

13. **方向键更新后光标复位到选中行**：`_rline` 的 `\n` 会把终端光标推到下一行，方向键/空格操作后加 `\033[%d;0H` 把光标移回 cursor 所在行行首，确保终端实际光标和 `▸` 指示的 app 在同一行。

14. **`q`/`b`/Enter 退出时跳到底部打印**：退出前 `\033[%d;0H\033[J` 跳到 TUI 摘要下方，清掉屏幕残留内容，避免退出信息打印到 TUI 中间造成显示混乱。

15. **`confirm()` 改为循环直到合法输入**：之前默认 Y 时 `[[ ! "$yn" =~ ^[Nn] ]]` 导致任何非 N 输入都通过（如 "DSD"）。改为 while 循环，只接受 Y/y/N/n/回车，非法输入重新询问。

16. **Bug 修复：`backup -y` 无效**：`auto_yes=1` 只跳过了 confirm 提示，但没有 `select_all_recommended`，导致 `has_any_selected` 为 false 直接退出。修复：auto_yes 模式下自动调用 `select_all_recommended`，跳过整个 TUI（包括 label/dest 提示），直接执行备份。

## Dockge 纯管理架构

- **根 `compose.yml` 精简为仅启动 Dockge**，不再 `include` 所有 stack
- **`stacks/*/.env` 改为符号链接** → `../../.env`，所有 stack 共享一份源文件（DRY 原则）
- **Docker Compose 支持 symlink 解析**：`docker compose config` 在子目录中正确展开 `${VAR}`
- **工作流**：`docker compose up -d` → 仅 Dockge → Web UI 导入各 stack → 启动/停止/管理
- 修改根 `.env` 后所有 stack 自动生效，无需额外同步步骤

- **mosquitto 启动失败**：`log_dest file /mosquitto/log/mosquitto.log` 但 `/mosquitto/log` 未挂载且 UID 1883 无写权限 → 解决：注释掉 file 日志，仅保留 stdout 输出

- **Dockge → Portainer 迁移**：Dockge 功能过于简陋，无法查看容器详情（网络/卷/环境变量）。创建 `stacks/portainer/`，移除 `dockge/` 目录，根 compose.yml 改为 include portainer
- **Portainer docker.sock 权限**：`user: 1000:1000` 导致容器无法访问 `/var/run/docker.sock`（docker 组 GID=989 不匹配）。Portainer 需要 root 运行（类似 mihomo），已移除 user 指令
- **权限例外规则**：需要系统级资源（docker.sock/NET_ADMIN/TUN）的应用不能降权，如 portainer、mihomo

- **新增 `stacks/openclaw/`**：基于官方文档 https://docs.openclaw.ai/install/docker
- 镜像 `ghcr.1ms.run/openclaw/openclaw:latest`（ghcr 国内镜像，与项目其他 ghcr 栈一致）；原版 `ghcr.io/openclaw/openclaw:latest`
- 端口 `18789`（控制 UI / 网关），`OPENCLAW_GATEWAY_BIND=lan` 局域网可访问
- 容器以 `node`(uid 1000) 运行，与项目 user 1000:1000 匹配；data/{config,auth} 已 chown 1000:1000
- 挂载 `./data/config:/home/node/.openclaw`、`/home/node/.config/openclaw`；额外挂 MEDIA_ROOT/DOWNLOAD_ROOT 供代理管文件
- 健康探针用 `node -e fetch(.../healthz)`（slim 镜像无 curl/wget）
- **首次启动实际步骤**（重要，已验证）：openclaw 不会自动生成配置/令牌，需手动初始化：
  1. 停止容器后运行 onboarding 交互向导会卡在非 TTY，改用 `config set`：`docker run --rm -u 1000:1000 -e HOME=/home/node -e OPENCLAW_CONFIG_DIR=/home/node/.openclaw -e OPENCLAW_CONFIG_PATH=/home/node/.openclaw/openclaw.json -v $PWD/data/config:/home/node/.openclaw <img> node dist/index.js config set --batch-json '[{"path":"gateway.mode","value":"local"},{"path":"gateway.bind","value":"lan"},{"path":"gateway.port","value":18789}]'`
  2. `lan` 模式**必须有鉴权**，否则 `Refusing to bind gateway to lan without auth`。生成令牌写入 `data/config/.env`（已被 .gitignore 忽略）：`openssl rand -hex 24` → `OPENCLAW_GATEWAY_TOKEN=...`，并在 compose 用 `env_file: - ./data/config/.env` 注入（只含令牌，不泄漏宿主机路径）
  3. 另需 `config set` 写入 `gateway.controlUi.allowedOrigins`（含 http://<NAS_IP>:18789），否则 LAN 其他机器浏览器访问 UI 被 CORS 拦截
  4. 探测：`curl http://<NAS_IP>:18789/healthz` 返回 `{"ok":true,"status":"live"}`，UI 根 `200`，容器 healthy
