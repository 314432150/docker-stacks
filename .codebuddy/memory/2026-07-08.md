## OpenClaw 代理修复 + 全局代理变量重构

- **问题**：OpenClaw 容器内 `HTTP_PROXY=http://127.0.0.1:7890` 指向容器自身而非宿主机，导致 npm 安装插件报 `ECONNREFUSED`。
- **原因**：`global.env` 中 `PROXY_HTTP=http://127.0.0.1:7890` 适合 host 模式容器，但 bridge 模式容器中 `127.0.0.1` 是容器自身 loopback。
- **修复**：`openclaw/compose.yml` 代理改为 `http://host.docker.internal:7890`（配合 `extra_hosts`）。

## 全局代理变量双套方案

将 `global.env` 拆分为两套，语义清晰：

```
PROXY_HOST=http://127.0.0.1:7890       # host 模式容器使用
PROXY_BRIDGE=http://host.docker.internal:7890  # bridge 模式容器使用
```

受影响文件：
- `global.env` — 拆分变量
- `jellyfin/compose.yml` — `${PROXY_HTTP}` → `${PROXY_HOST}`
- `openclaw/compose.yml` — 硬编码 → `${PROXY_BRIDGE}`
- `metatube/compose.yml` — 硬编码 → `${PROXY_BRIDGE}`
